# Garage Change Report Contract

Use this structure after meaningful work.

## Verdict

Choose exactly one:
- TEST
- CONTINUE
- NEEDS DECISION
- BLOCKED
- DONE

## Existing structure inspected

Name the relevant ownership chain and files.

## Files changed

List exact paths.

## What changed

Explain behavior and UI impact.

## What stayed the same

List important protected behavior.

## Verification

Report only what actually ran:
- build command and result
- tests and result
- simulator flow and result
- logs checked
- screenshots captured
- diff checks

## Residual risk

Name the one or two highest-value unknowns.

Avoid:
- vague "looks good"
- claiming visual verification from source inspection
- claiming tests passed when only build succeeded
- hiding environment blockers
