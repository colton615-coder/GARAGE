# BucketStrike Product Contract

## Purpose

BucketStrike generates and runs purposeful golf-practice sessions. It should feel like a guided training system, not a generic workout checklist.

## Primary journey

1. choose practice environment or training lane
2. inspect or select a plan
3. optionally edit the plan
4. start session
5. run drills in active practice
6. track execution according to drill mode
7. advance through the session
8. complete / reflect
9. retain a concise carry-forward memory

## Active-practice philosophy

The player is standing on a range, net, or putting green. Cognitive load matters.

Prefer:
- concise setup
- clear cue
- strong execution state
- obvious progress
- low-friction advancement
- minimal paperwork

Avoid:
- encyclopedic coaching text
- generic fault diagnosis on the execution screen
- manual tracking that adds no training value
- blocking advancement because a suggested timer is unfinished
- repeated nested cards
- equally weighted controls

## Teaching content

Player-facing drill teaching should answer:
- What is the goal?
- How do I set it up?
- What should I feel or focus on?
- What counts as execution?

Keep prescription logic, diagnostics, history analysis, and recommendation machinery separate from the live teaching surface unless the task explicitly requires them.

## Session memory

At session end, favor:
- one carry-forward cue
- a quick quality signal or rating

Do not turn reflection into a form-heavy workflow.
