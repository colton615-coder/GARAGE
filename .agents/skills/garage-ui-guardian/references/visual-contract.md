# Garage Visual Contract

## Core identity

Garage should feel:
- premium
- restrained
- dark
- precise
- performance-oriented
- instructional during training
- Apple-native in interaction and legibility

## Preferred visual ingredients

- black / deep green foundation
- emerald identity
- restrained gold accents
- glass used selectively for elevated or interactive surfaces
- thin dividers instead of unnecessary containers
- editorial spacing
- clear type hierarchy
- large hero visual only when it earns the space
- quiet secondary chrome

## Avoid

- card-on-card-on-card composition
- excessive corner-radius repetition
- every row inside its own floating panel
- multiple equal-weight primary buttons
- giant text that squeezes real content
- tiny instructional text
- generic dashboard tiles
- decorative charts with no training role
- AI-looking motivational filler
- too many badges
- large empty zones created only for aesthetics

## Instructional screen rule

During active practice, the user should understand:
1. what they are doing
2. how to set it up
3. the cue or feel
4. what to track
5. what action comes next

The visual hierarchy should answer those questions in that order unless the active mode requires the tracker or timer to dominate.

## Direction labels

When the user asks for design exploration, use these meanings:

### Apple
Native restraint, semantic hierarchy, quiet surfaces, system-feeling controls, polished motion.

### Sport-Tech
Sharper instrumentation, live feedback, high legibility, performance telemetry, strong execution state.

### Luxury
Deep material contrast, editorial type, subtle metallic accent, fewer but more deliberate surfaces.

Garage can blend the three, but every screen should have one dominant direction.
